Production

Preproduction
Episerver uses this environment to test production deployment, and verify performance and operational functionality. Customers may also use for UAT, load testing, or approved penetration testing. Preproduction scales automatically, and deployment is done by Episerver. Any content added in this environment will be overwritten during an upgrade. See the DXC Service Environments section on Episerver World for additional information.
 Available Available Available
Integration
Partners and customer deploy the full solution, as daily builds or continuous releases. Here customers can validate initial integrations with external systems, perform functional testing, and add initial content in the case of a first-time deployment. The integration environment has fixed configuration and no automatic scaling. You can deploy updates to Integration yourself. See the DXC Service Environments section on Episerver World for additional information.
Available Available Available
Additional deployment environments (max)
This is an Integration-like environment that is a replica of the primary Integration environment. The partner can deploy directly just like they would to the other standard Integration environment. However, the primary Integration environment in the DXC Service is considered the ""master"" and is the only one that can be used to follow the standard deployment flow to Preproduction/Production. The customer can deploy directly to the Additional Integration environment. When they are ready to deploy to Preproduction/Production they must first deploy to the primary Integration environment. This Additional Integration environment can never be used to deploy to Preproduction/Production. Common usage would be when there are multiple development streams running in parallel.
